# RAGFlow工作流配置文档

本文档详细说明如何在RAGFlow中创建用于提取"查询条件"和"查询内容"的智能体工作流。

---

## 一、工作流概述

### 功能说明
该工作流接收一个自然语言问题，通过LLM分析提取出两个字段：
- **查询条件**：问题中的定语、限定条件
- **查询内容**：问题中要查询的字段或信息

### 工作流结构
```
[开始节点] → [Agent节点] → [回答节点]
   Begin        Agent        Answer
```

---

## 二、创建工作流步骤

### 步骤1：进入RAGFlow工作流页面
1. 登录RAGFlow管理界面
2. 点击左侧菜单「智能体」或「Agent」
3. 点击「创建智能体」按钮
4. 选择「工作流编排」模式

### 步骤2：配置开始节点（Begin）

开始节点通常已自动创建，保持默认配置即可。

**可选配置：**
- 开场白：`请输入要分析的问题`

### 步骤3：添加Agent节点

从左侧组件面板拖拽「Agent」节点到画布。

#### 3.1 基本配置

| 配置项 | 值 |
|--------|-----|
| 节点名称 | QA分类提取 |
| 模型 | Qwen3-32B（或其他可用模型） |

#### 3.2 模型参数

| 参数 | 值 | 说明 |
|------|-----|------|
| Temperature | 0.01 | 低温度保证输出稳定 |
| Top P | 0.8 | |
| Max Tokens | 1024 | 足够返回JSON |
| 历史消息窗口 | 0 | 不需要历史上下文 |

#### 3.3 系统提示词（System Prompt）

```
## 角色定义
你是一个专业的SQL问题分析器，负责从用户问题中提取"查询条件"和"查询内容"。/no_think

## 任务说明
从用户的自然语言问题中，准确提取两个字段：
1. **查询条件**：问题中的定语、限定条件、筛选条件
2. **查询内容**：问题中要查询的具体字段或信息

## 提取规则

### 查询条件的识别
查询条件是用来限定查询范围的部分，通常包含：
- 时间条件：2024年、本月、去年等
- 数值条件：超过100%、低于70、大于50等
- 等值条件：xxx为yyy、xxx是yyy、xxx等于yyy
- 范围条件：在xxx到yyy之间
- 分组条件：按xxx统计/分组
- 全量标识：所有xxx（注意：如果是"查询所有A的B"，条件应为"所有A"而非仅"所有"）

### 查询内容的识别
查询内容是用户真正想获取的信息，通常是：
- 具体字段：设备编号、合同金额、健康指数等
- 信息类型：设备信息、合同信息、统计数量等
- 参数数据：供应商参数、技术参数等

## 关键规则
1. "查询所有A的B" → 条件="所有A", 内容="B"
2. "查询A为xxx的B" → 条件="A为xxx", 内容="B信息"
3. "查询所有A信息" → 条件="所有", 内容="A信息"
4. "统计xxx" → 条件="", 内容="xxx"
5. "按A统计B" → 条件="按A", 内容="B"

## 输出格式

```

#### 3.4 用户提示词模板

在「提示词」或「Prompts」配置区域添加：

```
请分析以下问题，提取查询条件和查询内容：

{question}
```

**注意**：`{question}` 是占位符，会被实际的用户输入替换。

### 步骤4：添加回答节点（Answer）

从左侧组件面板拖拽「Answer」节点到画布。

保持默认配置即可，该节点会将Agent的输出返回给调用方。

### 步骤5：连接节点

用连线将节点按以下顺序连接：
```
Begin → Agent(QA分类提取) → Answer
```

### 步骤6：保存并发布

1. 点击「保存」按钮
2. 点击「发布」按钮使工作流生效

## 三、获取Agent ID

工作流创建完成后，需要获取Agent ID用于API调用：

### 方法1：从URL获取
1. 在工作流编辑页面，查看浏览器地址栏
2. URL格式通常为：`http://xxx/agent/{agent_id}/...`
3. 复制其中的 `agent_id` 部分

### 方法2：从API获取
1. 进入「智能体」列表页面
2. 找到刚创建的工作流
3. 点击「详情」或「设置」
4. 在详情页面可以看到Agent ID

### 方法3：查看Agent列表API
```bash
curl -X GET "http://your-ragflow-server/api/v1/agents" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## 五、测试工作流

### 在RAGFlow界面测试

1. 进入工作流编辑页面
2. 点击「测试」或「调试」按钮
3. 输入测试问题，例如：
   - `查询所有预算的预算编码`
   - `查询设备名称为北斗授时服务器的设备卡供应商参数`
4. 验证输出是否为正确的JSON格式

### 使用Python测试

```python
import requests

base_url = "http://192.168.110.147:8888"
api_key = "ragflow-xxxxx"
agent_id = "your-agent-id"

response = requests.post(
    f"{base_url}/api/v1/agents/{agent_id}/completions",
    headers={
        "Authorization": f"Bearer {api_key}",
        "Content-Type": "application/json"
    },
    json={
        "question": "查询所有预算的预算编码",
        "stream": False
    }
)

print(response.json())
```

期望输出包含：
```json
{"查询条件": "所有预算", "查询内容": "预算编码"}
```

## 六、常见问题

### Q1: 输出不是JSON格式
**解决方案**：
- 确保系统提示词中包含 `/no_think` 标记（关闭思考模式）
- 将Temperature设置为0.01
- 在系统提示词末尾强调"只返回JSON，不要有任何解释"

### Q2: 查询条件提取不准确
**解决方案**：
- 检查系统提示词中的示例是否覆盖了该类型的问题
- 添加更多相关示例到系统提示词中

### Q3: API调用超时
**解决方案**：
- 增加脚本中的 `timeout` 配置
- 检查RAGFlow服务器负载

### Q4: Agent ID找不到
**解决方案**：
- 确保工作流已保存并发布
- 尝试从API列表中获取

## 七、完整工作流配置参考

以下是完整的工作流JSON配置（可导入RAGFlow）：

```json
{
  "name": "QA分类提取工作流",
  "description": "从自然语言问题中提取查询条件和查询内容",
  "components": {
    "begin": {
      "params": {
        "prologue": "请输入要分析的问题"
      }
    },
    "agent_qa_extract": {
      "component_name": "Agent",
      "params": {
        "llm_id": "Qwen3-32B",
        "temperature": 0.01,
        "top_p": 0.8,
        "max_tokens": 1024,
        "message_history_window_size": 0
      },
      "sys_prompt": "（见上文系统提示词）",
      "prompts": [
        {
          "role": "user",
          "content": "请分析以下问题，提取查询条件和查询内容：\n\n{question}"
        }
      ]
    },
    "answer": {
      "params": {}
    }
  },
  "graph": {
    "edges": [
      {"source": "begin", "target": "agent_qa_extract"},
      {"source": "agent_qa_extract", "target": "answer"}
    ]
  }
}
```
